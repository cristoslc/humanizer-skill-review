# User Experience

This workspace has no UI. "Users" are human reviewers and the agents assisting them.

## UX principle: low-friction review

The workspace should make it easy to:
1. Fetch a candidate skill (one command).
2. Orient quickly (read the profile, not the full source).
3. Compare side-by-side (the matrix is the primary artifact).
4. Record a decision without ambiguity (ADR template with context/options/rationale).

## Accessibility

All artifacts are plain markdown. No charts requiring sight, no color-coded tables that fail in monochrome. Mermaid diagrams are used sparingly and always have a text alternative in the surrounding prose.
