## Artifacts

- [`rubric.md`](./rubric.md) — scoring criteria (10 dimensions, numeric 1–3 scale)
- [`matrix.md`](./matrix.md) — living scoring table, one row per candidate; auto-generated by `tests/harness/build-matrix.py` from `tests/results/*/scores.json`
- [`profiles/`](./profiles/) — per-candidate profiles (`<short-name>.md`), one per candidate; start from [`profile-template.md`](./profile-template.md)
- [`profile-template.md`](./profile-template.md) — template for per-candidate profiles
- [`recommendation.md`](./recommendation.md) — final synthesis

## How scores get here

```
scripts/fetch-candidates.sh  →  tests/harness/exercise-skill.sh
                              →  tests/harness/score-outputs.py
                              →  tests/harness/build-matrix.py  →  comparison/matrix.md
```

The matrix is auto-generated. Do not edit it by hand — run `./scripts/run-all.sh` to regenerate.

## Lifecycle

- `rubric.md` is mutable — version it when criteria change.
- `profiles/<name>.md` are mutable — update as candidates release new versions; note the SHA reviewed.
- `matrix.md` is auto-generated and overwritable.
- `recommendation.md` is mutable — synthesized once enough candidates have ADRs.
